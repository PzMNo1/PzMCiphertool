  模块定位
  新增侧边栏入口：Skill/MCP 实验室

  Registry、社区目录，并提供搜索、筛选、复制配置、跳转文档、风险提示、收藏/待接入清
  单。

  - 操作按钮用 .cyber-button
  - 搜索框沿用现有输入框样式

  3. Skill 库

  卡片内容建议
  每个资源卡统一字段：

  名称
  类型：MCP Registry / MCP Server / Skill Registry / Plugin Marketplace
  推荐程度：官方优先 / 社区可参考 / 需审查
  适用场景：搜索、数据库、浏览器、文件、设计、知识库、代码工具等
  入口按钮：官网 / 文档 / GitHub / 复制配置
  风险提示：是否需要本地权限、API Key、shell、文件访问

  整体规划位置
  第一阶段原计划是：

  1. 静态目录 + 搜索 + 外链 + 复制配置
  2. 收藏、标签、风险评分、本地 JSON 数据源
  3. 接入后端或 Agent Runtime，允许真正测试 MCP Server
  4. 做安装向导，按 Codex / Claude / ChatGPT / Cursor 分类生成配置

  现在处于 第 1 阶段完成，已经开始触碰第 2 阶段前半部分。
  因为收藏、标签、风险筛选已经做了，但还没有做到“独立数据源、真实评分、资源维护机制”。

  和社区 Hub 的差距
  对比 Smithery、Glama、PulseMCP、Docker MCP Catalog、MCP Atlas、FindSkills 这些站，目
  前差距主要是：

  - 没有实时数据源：现在资源是写死在 JS 里的，不能自动更新。
  - 没有详情深度：社区 Hub 通常能看到工具列表、schema、README、安装命令、包来源、更新
    时间。
  - 没有真实可用性检测：不能判断链接是否失效、MCP 是否能连通、工具是否能调用。
  - 没有权限模型：目前只是文字风险提示，还没有结构化记录“文件读写、shell、网络、数据
    库、浏览器”等权限。
  - 没有评分系统：没有维护度、可信度、权限风险、文档质量、安装复杂度评分。
  - 没有安装向导：现在只复制模板，不会按 Codex/Claude/ChatGPT/Cursor 生成具体配置。
  - 没有导入/导出：收藏清单不能导出为 JSON，也不能从外部列表导入。
  - 没有社区贡献机制：不能由用户添加、编辑、审核资源。
  - 没有真正 MCP Inspector：不能在页面里查看 server tools/schema，也不能发起测试调用。

  下一步建议
  我建议不要直接追求“大而全 Hub”，先做适合这个项目的本地实验室路线：

  1. 第二阶段 A：数据结构化
      - 把资源从 JS 内部数组拆成 mcpskilllab/resources.js 或 resources.json
      - 每个资源增加字段：
        platforms、permissions、installModes、authRequired、maintenance、trustScore、
        lastChecked
      - UI 增加权限筛选：文件、Shell、网络、浏览器、数据库、API Key
  2. 第二阶段 B：本地 Hub 能力
      - 收藏清单导入/导出 JSON
      - 用户自定义新增资源，先存在 localStorage
      - 给资源加备注：为什么收藏、准备接入到哪里、当前状态
  3. 第三阶段：安装向导
      - 选择平台：Codex / Claude / ChatGPT / Cursor / 通用 MCP
      - 根据平台生成配置片段
      - 给出“本地运行 / 远程 MCP / Docker MCP”的不同模板
      - 明确哪些需要 API Key，哪些需要网络，哪些会跑命令
  4. 第四阶段：检测与评分
      - 链接可达性检测
      - GitHub repo 活跃度字段
      - 权限风险评分
      - 安装复杂度评分
      - 文档完整度评分
  5. 第五阶段：真正实验室
      - 接入后端或 Agent Runtime
      - 对远程 MCP 做连接测试
      - 展示 tools/schema
      - 支持一次安全的 dry-run 调用
